## 一、工具核心定位：解决RAW文件的「预览+解析」刚需
对于经常接触RAW格式（CR3/CR2/NEF/ARW/DNG等）的用户来说，肯定遇到过这些问题：
- RAW文件体积大，普通图片查看器无法直接预览；
- 想快速提取嵌入的预览图，却需要复杂的专业软件；
- 需要查看拍摄参数（相机型号、快门、光圈等），却找不到轻便的解析工具；
- 提取的预览图丢失原始EXIF信息，影响后续整理。

## 二、核心功能亮点
### 1. 全格式兼容，覆盖主流RAW类型
工具支持目前市面上绝大多数RAW格式：CR3/CR2/NEF/ARW/DNG/RAF/RW2/ORF/3FR/IIQ等，无论是佳能、尼康、索尼、富士等品牌相机拍摄的文件，都能轻松解析。

### 2. 智能提取嵌入预览图，自动格式兼容
- 自动识别RAW文件中嵌入的所有预览图（通常包含缩略图、标准预览图、高清预览图）；
- 针对TIFF等浏览器不兼容格式，自动转换为JPEG/WebP，确保前端正常显示；
- 保留原始格式预览图下载选项，满足专业用户需求；
- 提取的预览图会完整嵌入原始RAW文件的EXIF信息（相机型号、镜头参数、拍摄时间等），无需额外手动添加。

### 3. 完整EXIF信息解析，参数一目了然
解析并展示RAW文件的核心信息，分类清晰：
- 文件信息：文件名、大小、拍摄时间、格式；
- 相机信息：相机型号、镜头型号、传感器尺寸、分辨率；
- 曝光参数：ISO感光度、快门速度、光圈、焦距、曝光补偿、白平衡。
所有EXIF信息自动截断过长字段，保证页面加载流畅，同时不丢失关键内容。

### 4. 安全高效，无后顾之忧
- 支持最大200MB的RAW文件上传，满足绝大多数专业需求；
- 文件上传后10分钟自动清理，保护用户数据隐私；
- 纯网页端操作，无需安装任何插件或客户端软件；
- 流式传输预览图，避免内存溢出，加载速度更快。

## 三、部署过程简易说明
作为开发者，这款工具的部署门槛较低，适合个人或小型团队快速上线：
1. 环境依赖：Python 3.8+、Flask框架、exiftool（核心解析工具）、PIL（图片处理）；
2. 服务器要求：建议2GB内存以上，支持Docker部署（我采用Docker容器化部署，稳定性更高）；
3. 部署步骤：
   - 克隆代码仓库，安装依赖包（flask、apscheduler、pillow等）；
   - 安装exiftool（Ubuntu用apt install，Mac用brew install）；
   - 配置端口（默认10099），启动Flask服务；
   - 配置Nginx反向代理，绑定域名（支持HTTPS）；
4. 稳定性优化：添加定时任务自动清理过期文件，避免磁盘占用过高。
### 1. 安装Python依赖

```bash
# 进入项目目录

# 安装依赖（使用项目虚拟环境）

#### 1. Ubuntu/Debian系统

```

####  macOS系统

```bash

```

如果有大量RAW文件需要提取预览图用于快速筛选，无需逐一用Lightroom等软件打开，通过工具批量上传（后续将支持批量上传功能），就能快速获取所有预览图，大大提升工作效率。

提取的预览图会完整保留原始RAW文件的EXIF信息，下载后用于归档、分享时，不会丢失关键的拍摄数据，对专业影像管理非常友好。
